# Harun Farocki｜EYE/MACHINE MONTAGE / 双屏观看机制（案例级）

> 关键气质：**人眼 vs 机器眼**。  
> 核心方法：把图像拆成两套阅读：叙事视线与操作视线并置。

## 0) 三件套
1. **双屏主视觉（Split Screen）**：左 HUMAN VIEW / 右 OPERATION VIEW
2. **注释条目**：左右屏各4条（可验证）
3. **观看条件清单**：6条（距离、角度、分辨率、遮挡）

## 1) 归属牌组
- **拆机系｜双屏对照台牌组（Eye/Machine）**

## 2) Director_spec
- 语气：`冷静教学 / 工程审计 / 讽刺`
- 对照：`信息对照 / 权力对照 / 失明点对照`
- 风险：`不指控具体个人；只讨论机制`

## 3) Visual_spec
- 左屏保留“人类叙事”构图
- 右屏做“机器视线”叠层：框选、阈值线、编号、热点
- 避免给出可用于跟踪的精确位置/身份信息

## 4) Artifacts
1. **左右注释条目**（各4）
2. **观看条件清单**（6）
3. **机制结论句**（≤22字）
4. **缺口声明**（我们看不到什么）

## 5) Workflow
1. 识别：监控感、系统界面、流程场景、可被“操作化阅读”  
2. 配牌：双屏对照台为主；辅以“不可见系统板/系统脚注页/证据墙”  
3. 导演：默认教学；强讽刺需降风险  
4. 生成：先做右屏叠层，再写注释  
5. 交付：附免责声明“分析机制非指控”
