# Hito Steyerl｜POOR IMAGE COMPILER / 平台皮肤编译器（案例级）

> 关键气质：**压缩、再上传、流通痕迹就是内容**。  
> 核心方法：把图像变成“平台皮肤”，并把损失/增益写成收据。

## 0) 三件套（输出骨架）
1. **poor image 主视觉（Platform Skin）**：再上传痕迹拼贴、低清、边缘锯齿、压缩噪点
2. **来源链（Source Chain）**：A→B→C（每跳的损失/增益）
3. **压缩收据（Compression Receipt）**：字段化“画质/码率/水印/裁切/时间戳”变化

## 1) 归属牌组
- **皮肤系｜平台皮肤页牌组（Platform Skin / Poor Image）**

## 2) Director_spec
- 语气：`讲解型 / 冷嘲型 / 学术脚注型`
- 结构：`先皮肤后来源 / 先来源后皮肤`
- 透明度：`明确虚构链 / 模拟真实链`
- 风险：`不点名平台 / 可点名平台（需风控）`

## 3) Visual_spec
- poor image 主视觉：
  - 低清拼贴，多层水印、时间戳、转发框、播放条、加载图标
  - 画面可见压缩、马赛克、重采样、截图边框
- Context Shot（可选）：
  - 装置化放映现场、投影/屏幕墙、观众背影

## 4) Artifacts
1. **来源链A→B→C**（每跳：损失/增益各3条）
2. **平台动词小论文**（160-220字，动词开头：压缩/裁切/转发/加速…）
3. **压缩收据**（字段：分辨率、裁切、叠字、水印、噪点、伪色、字幕）
4. **“看不清”带来的政治性清单**（5条）

## 5) Workflow
1. 识别：截图边框、时间戳、水印、低清噪点、社交UI  
2. 配牌：平台皮肤页为主；辅以“档案蒙太奇/双屏对照台/系统脚注页”  
3. 导演：默认不点名平台；若点名需降锋利度或加免责声明  
4. 生成：先做皮肤，再做来源链和收据  
5. 交付：附“模拟件”声明（避免误导为真实传播链）
