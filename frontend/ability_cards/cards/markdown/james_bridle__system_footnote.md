# 案例能力牌：SYSTEM FOOTNOTE / 系统脚注板（James Bridle 向）

## 1) 归属牌组（固定掉落载体）
**牌组：证据板牌组（System Annotation + Plain Layer）**  
固定掉落（不可漂移）：
- 标签注释图（6标签）
- 白话解释 90–120字
- 合法行动建议 1条
- 网页长图存在场景

---

## 2) director_spec（0–2 个按钮 + 默认路径）
- 按钮1：系统：算法/平台（默认）/ 城市基础设施
- 按钮2：重心：注释图（默认）/ 白话页
- 默认：算法 + 注释图

---

## 3) visual_spec（主视觉 Image + 存在场景 Context Shot）

### 3.1 主视觉（Image）提示词
```text
在照片上叠6标签：#阈值 #分类 #路径 #遗漏 #误差 #反馈回路（指向具体细节）；右侧白话解释90–120字（不用行话）；底部1条合法行动建议（识别/拒绝/记录）。
```

### 3.2 存在场景（Context Shot）提示词
```text
白底讲解板或网页长图：上半注释图，下半白话与建议；不出现真实机构名。
```

---

## 4) artifacts（主展品候选 2–4：名称 + 详细提示词）
### Artifact A（默认）｜System Tags 6
```text
6条：线索→系统动作→牺牲的信息。
```
### Artifact B｜Plain Explainer
```text
90–120字：解释系统如何分类/推荐/审查；结尾一个具体提问。
```
### Artifact C｜Action Suggestion
```text
1条合法建议：识别信号→选择策略（拒绝/延迟/转移/记录）。
```

---

## 5) 一次完整 Session（Scan → Load → Direct → Reveal → Collect）
Scan：系统在工作 → Load：系统类型 → Direct：2按钮 → Reveal：注释+白话+建议 → Collect：主展品=标签。

---

## 6) 安全与边界（必须遵守）
- 不提供技术攻击/绕过步骤。
- 不输出可定位信息。
- 建议必须合法、低风险。
