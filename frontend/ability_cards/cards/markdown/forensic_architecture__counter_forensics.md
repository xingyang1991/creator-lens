# Forensic Architecture｜COUNTER-FORENSICS / 证据板编译器（案例级）

> 关键气质：**可验证、可反驳、强调不确定性**。  
> 核心方法：把单张图像转为“可复核的证据板”（观察/推断/缺失三分法 + 反证点）。

## 0) 三件套（输出骨架）
1. **证据板成片（Evidence Board）**：6点标注 + 2个放大窗 + 关键线索编号  
2. **现场/听证会陈列照（Context Shot）**：展墙/桌面证据墙被观看的照片  
3. **不确定性日志页（Uncertainty Log）**：我们不知道什么、需要哪些外部证据、如何推翻自己

## 1) 归属牌组
- **卷宗系｜证据墙牌组（Evidence Wall / Counter-Forensics）**

## 2) Director_spec（可按钮）
- 语气：`冷静研判 / 法庭式克制 / 公开听证口吻`
- 结构：`时间线优先 / 空间定位优先 / 物证链优先`
- 强度（危险档位）：`中性研判（默认） / 指控叙事（Danger Slot）`
- 透明度：`证据来源清晰 / 允许猜测但必须标“推断” / 只写观察`

## 3) Visual_spec（图像生成提示）
- 主视觉（Evidence Board）：
  - 白底或深灰底调查板，编号贴纸、箭头连线、红色圈注（低饱和），2个放大窗，清晰排版
  - 避免真实组织logo；避免精确地理定位；人脸默认模糊/遮挡
- Context Shot：
  - 白盒子展厅/会议室/听证会现场，证据板挂墙或桌面摊开，观众背影

## 4) Artifacts（结构化文本产物）
1. **T-2…T+2 时间线**（含缺口）
2. **观察 / 推断 / 缺失**三栏表（至少各6条）
3. **反证点清单**（至少4条：如何证明我们错了）
4. **不确定性日志**（3段：未知、需要、风险）

## 5) Workflow（系统使用流程）
1. 识别：截图/新闻图/监控感/标注潜力/地图或UI痕迹  
2. 配牌：证据墙为主，辅以“系统脚注页/不可见系统板/全景放大窗”（跨卡组混配）  
3. 导演：默认“中性研判”，只有在风险评估通过才允许“指控叙事”按钮  
4. 生成：先出证据板主视觉，再出日志与反证点  
5. 交付：附带免责声明与去识别清单
